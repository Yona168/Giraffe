GServer

class GServer : Messenger, Server

Platform and version requirements: JVM

The given implementation for Server. This class uses a MutableMap to store client UUIDs to the Client objects, which represent server-side connections. CoroutineScope.coroutineContext is overriden with Dispatchers.IO+Messenger.job. After accepting a Client, it writes a handshake packet (see GClient description), giving it a session UUID as referenced by Client.sessionUUID. Any Client that uses this server should handle the packet accordingly, or ignore it.

Parameters

address - The SocketAddress to have the server run on

packetProcessor - The PacketProcessor to process received packets with.

pool - The Pool to get empty packets from.
